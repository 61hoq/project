# DSJA 불법 광고 데이터 분석 코드 설명

작성일: 2026-05-06

## 1. 코드의 전체 목적

이 코드는 여러 사이트에서 수집한 게시글 데이터를 하나로 합친 뒤, 게시글 제목을 정리하고, 불법 광고 유형을 자동 분류하며, 분류 결과를 바탕으로 워드클라우드, 지역 분포, 텔레그램 아이디 네트워크 등을 분석하는 흐름입니다.

코드는 다음 순서로 작동합니다.

1. Google Drive에 저장된 크롤링 CSV 파일을 불러옵니다.
2. 여러 CSV를 하나로 합치고 중복 URL을 제거합니다.
3. 제목, 날짜, 조회수 같은 기본 정보를 정리합니다.
4. 특수문자와 변형 문자를 일반 문자로 바꿉니다.
5. 불법 키워드 사전과 지역명 사전을 불러옵니다.
6. 게시글 제목에서 의미 있는 명사만 추출합니다.
7. 키워드 규칙과 모델 예측을 함께 사용해 불법 광고 카테고리를 정합니다.
8. 분류된 데이터를 저장합니다.
9. 저장된 데이터를 이용해 시각화와 추가 분석을 수행합니다.

## 2. 데이터 입력과 통합

코드는 Colab 환경에서 Google Drive를 마운트한 뒤 `/content/drive/MyDrive/크롤링 결과/` 아래의 데이터를 사용합니다.

| 항목 | 설명 |
| --- | --- |
| `크롤링 완료` | 일반적으로 수집된 CSV 파일 폴더 |
| `크롤링 완료(1만개 제한)` | 사이트별 수집 개수 제한이 걸린 CSV 파일 폴더 |
| `전국행정동리스트.xlsx` | 지역명 매핑에 사용하는 행정동 데이터 |
| Google Sheets 키워드 사전 | 불법 광고 카테고리별 키워드 목록 |

CSV 파일은 인코딩이 파일마다 다를 수 있으므로 `utf-8-sig`, `cp949`, `euc-kr` 순서로 읽기를 시도합니다. 이후 모든 파일을 합치고 URL 기준으로 중복 게시글을 제거합니다.

`1만개 제한` 폴더에서 온 데이터에 해당하면 `is_truncated=True`로 표시합니다. 이는 해당 사이트의 데이터가 전체가 아니라 일부만 수집되었을 가능성을 기록하기 위한 값입니다.

## 3. 기본 전처리

통합 데이터에서 주로 정리하는 컬럼은 다음과 같습니다.

| 컬럼 | 처리 내용 |
| --- | --- |
| `view` | 숫자로 변환하고, 비어 있으면 0으로 처리 |
| `date` | `26-02-13` 같은 날짜를 `2026-02-13` 형태로 보정 |
| `title` | 특수문자 등을 정리해 `title_cleaned` 생성 |
| `url` | 중복 제거 기준으로 사용 |
| `site` | 어느 사이트에서 데이터를 수집했는지 표시 |

불법 광고 제목에는 일반 문자처럼 보이지만 실제로는 특수 유니코드인 글자들이 많이 들어갑니다. 코드는 `MANUAL_MAP`과 `unicodedata.normalize('NFKC')`를 사용해 이런 문자를 원형 문자에 가깝게 바꿉니다.

## 4. 키워드 사전과 지역명 사전

코드는 두 종류의 외부 사전을 사용합니다.

| `불법키워드` | 불법 광고 유형과 대표 키워드 | 카테고리 분류 기준 |
| `전국행정동리스트.xlsx` | 전국 행정구역명 | 지역명 추출 및 지역 분석 기준 |

이 파일은 코드에서 `df_kw`라는 데이터프레임으로 불러와 사용됩니다. 원래 한 셀 안에 여러 키워드가 쉼표로 묶여 있기 때문에, 코드는 이를 한 줄에 하나의 키워드가 오도록 펼칩니다. 이후 공백을 제거하고, `NFKC` 정규화를 적용해 비슷하게 보이는 문자들을 가능한 한 같은 형태로 맞춥니다.

이 키워드 사전은 세 가지 목적에 사용됩니다.

1. 형태소 분석기 사용자 사전에 등록해 중요한 키워드가 잘게 쪼개지지 않도록 합니다.
2. 게시글 제목에서 추출된 명사와 키워드가 직접 일치하는지 확인합니다.
3. 나이브 베이즈 1차 분류 모델의 학습 데이터를 만드는 기준으로 사용합니다.


지역명 사전은 게시글 제목에 등장하는 지역명을 식별하고 정리하기 위한 지역명 사전입니다. 파일에는 `키워드추출기`라는 시트가 있으며, 실제 데이터는 `대분류`, `시 / 군`, `구`, `동/면/리` 구조로 구성되어 있습니다.

총 19,832개의 행정구역 데이터 행이 들어 있습니다. 시도 단위는 17개이며, 시군 단위 고유값은 158개, 구 단위 고유값은 77개, 동/면/리 단위 고유값은 10,362개입니다.

파일의 기본 구조는 다음과 같습니다.

| 컬럼 | 의미 | 예시 |
| --- | --- | --- |
| 대분류 | 광역 시도 | 서울특별시, 경기도, 제주특별자치도 등 |
| 시 / 군 | 시 또는 군 | 수원시, 서귀포시 등 |
| 구 | 자치구 또는 일반구 | 종로구 등 |
| 동/면/리 | 세부 행정구역 | 청운동, 신교동, 하천리 등 |

포함된 광역 시도는 다음 17개입니다.

```text
강원도, 경기도, 경상남도, 경상북도, 광주광역시, 대구광역시, 대전광역시,
부산광역시, 서울특별시, 세종특별자치시, 울산광역시, 인천광역시,
전라남도, 전라북도, 제주특별자치도, 충청남도, 충청북도
```

이 파일은 코드에서 `df_admin`으로 불러와 사용됩니다. 코드는 이 데이터를 바탕으로 `시도`, `시군`, `구`, `동면리` 컬럼을 만들고, 지역명을 중분류와 소분류 형태로 정리합니다. 예를 들어 게시글 제목에 특정 동 이름이나 구 이름이 등장하면, 그 단어가 실제 행정구역명인지 확인하고 어느 시도나 시군구에 속하는지 연결하는 데 사용됩니다.

이 지역명 사전은 두 가지 목적에 사용됩니다.

1. 형태소 분석기 사용자 사전에 지역명을 등록해 지명이 분리되지 않도록 합니다.
2. 게시글 제목에서 지역명을 찾아 `region_list` 같은 분석용 컬럼을 만드는 데 사용합니다.

불법 광고 게시글은 특정 지역명과 함께 노출되는 경우가 많습니다. 따라서 이 파일은 단순히 주소를 정리하기 위한 자료가 아니라, 불법 광고가 어떤 지역명과 함께 등장하는지 분석하기 위한 기반 데이터입니다.

## 5. 형태소 분석과 명사 추출

게시글 제목 전체를 그대로 분류에 사용하지 않고, 의미 있는 단어만 추출하기 위해 `kiwipiepy`의 `Kiwi` 형태소 분석기를 사용합니다.

이 단계의 핵심은 다음과 같습니다.

1. 키워드 사전과 지역명 사전에 있는 단어를 Kiwi 사용자 사전에 등록합니다.
2. 사전에 있는 중요한 단어가 형태소 분석 중에 잘게 쪼개지지 않도록 합니다.
3. 제목에서 명사 중심의 단어를 추출해 `nouns` 컬럼에 저장합니다.

## 6. 1차 분류: 나이브 베이즈 모델

코드는 먼저 비교적 가벼운 모델인 나이브 베이즈 분류기를 사용합니다. 이 모델은 각 카테고리의 키워드 사전을 바탕으로 “어떤 단어 조합이 어떤 카테고리에 자주 등장하는지”를 학습합니다.

| 설정 | 값 | 의미 |
| --- | --- | --- |
| `TARGET_N` | 2000 | 카테고리당 2,000개의 학습 샘플 생성 |
| 난수 시드 | 42 | 같은 결과가 재현되도록 고정 |
| 단어 조합 | unigram, bigram | 단일 키워드와 두 키워드 조합 사용 |
| 벡터화 방식 | `CountVectorizer` | 단어 등장 횟수를 숫자로 변환 |
| `ngram_range` | `(1, 2)` | 한 단어와 두 단어 조합을 함께 사용 |
| 분류기 | `MultinomialNB` | 단어 빈도 기반 분류 모델 |

모델은 각 게시글의 `nouns`를 보고 가장 가능성이 높은 카테고리 1순위와 2순위를 계산합니다.

| 컬럼 | 의미 |
| --- | --- |
| `top1_cat` | 모델이 가장 가능성 높다고 본 카테고리 |
| `top1_conf` | 1순위 카테고리에 대한 확신도 |
| `top2_cat` | 두 번째로 가능성 높은 카테고리 |
| `top2_conf` | 2순위 카테고리에 대한 확신도 |

## 7. 최종 분류 방식: 규칙과 모델 결합

최종 카테고리는 단순히 모델 예측값만 쓰지 않습니다. 키워드 사전과 모델 예측을 함께 사용합니다.

| 단계 | 판단 방식 | 설명 |
| --- | --- | --- |
| Tier 1 | 키워드 기반 확정 | 한 카테고리의 키워드가 뚜렷하게 매칭되면 그 카테고리로 분류 |
| Tier 2 | 키워드 + 모델 | 여러 카테고리 키워드가 비슷하게 매칭되면 모델 예측으로 결정 |
| Tier 3 | 모델 기반 | 키워드가 없지만 모델 확신도가 충분하면 모델 예측 사용 |
| Tier 3 (Margin) | 모델 기반 보완 | 1순위와 2순위 차이가 충분하면 모델 예측 사용 |
| Tier 4 | 미분류 | 키워드도 약하고 모델 확신도도 낮은 경우 |

모델 기반 판단 기준은 다음과 같습니다.

| 조건 | 처리 |
| --- | --- |
| `top1_conf >= 0.25` | 1순위 모델 예측을 사용 |
| `top1_conf >= 0.18`이고 `top1_conf - top2_conf >= 0.10` | 1순위와 2순위 차이가 충분하다고 보고 사용 |
| 위 조건을 만족하지 않음 | Tier 4, 즉 미분류 처리 |

`연락수단(텔레그램)`은 일반 카테고리와 조금 다르게 처리합니다. 게시글의 불법 유형이 아니기 때문에, `is_contact` 같은 별도 표시값으로 관리합니다.

## 8. 2차 모델 학습: KLUE-RoBERTa

코드는 나이브 베이즈와 규칙 기반으로 정리된 분류 결과 중 `Tier 4`를 제외한 데이터를 사용해 한국어 딥러닝 언어모델인 `klue/roberta-base`를 추가 학습합니다.

### 학습 데이터 분할

| 항목 | 값 |
| --- | --- |
| 사용 데이터 | `logic_tier != 'Tier 4'`인 데이터 |
| 입력 컬럼 | `title_cleaned` |
| 정답 컬럼 | `final_category`를 숫자로 바꾼 `label` |
| 분할 방식 | Train / Validation / Test |
| 비율 | 8 : 1 : 1 |
| `test_size` 1차 | 0.2 |
| `test_size` 2차 | 0.5 |
| `random_state` | 42 |
| `stratify` | 카테고리 비율이 유지되도록 label 기준 적용 |

### RoBERTa 학습 하이퍼파라미터

| 항목 | 코드 값 | 설명 |
| --- | --- | --- |
| 사전학습 모델 | `klue/roberta-base` | 한국어 RoBERTa 기반 모델 |
| 최대 토큰 길이 | `MAX_LEN = 64` | 제목을 최대 64 토큰까지 사용 |
| 배치 크기 | `BATCH_SIZE = 32` | 한 번에 32개 샘플씩 학습 |
| 에폭 수 | `EPOCHS = 5` | 전체 학습 데이터를 최대 5번 반복 |
| 학습률 | `LR = 2e-5` | 모델 가중치를 업데이트하는 속도 |
| 옵티마이저 | `AdamW` | 딥러닝 모델 학습에 자주 쓰는 최적화 방식 |
| 가중치 감쇠 | `weight_decay = 0.01` | 과적합을 줄이기 위한 규제값 |
| 손실 함수 | `CrossEntropyLoss` | 다중 카테고리 분류용 손실 함수 |
| 라벨 스무딩 | `label_smoothing = 0.1` | 모델이 한 정답에 과하게 확신하지 않도록 조정 |
| 스케줄러 | `get_linear_schedule_with_warmup` | 학습률을 학습 단계에 맞춰 조절 |
| 워밍업 스텝 | `total_steps // 10` | 전체 학습 스텝의 10% 동안 서서히 학습률 증가 |
| 총 학습 스텝 | `len(train_loader) * EPOCHS` | 학습 배치 수와 에폭 수로 계산 |
| 그래디언트 클리핑 | `1.0` | 학습 중 값이 너무 크게 튀는 것을 방지 |
| 조기 종료 patience | `2` | 검증 손실이 2번 연속 개선되지 않으면 중단 |
| 디바이스 | CUDA 가능 시 GPU, 아니면 CPU | Colab GPU 사용 가능 |
| DataLoader worker | `num_workers = 2` | 데이터 로딩 보조 프로세스 수 |
| 학습 데이터 셔플 | Train만 `shuffle=True` | 학습 순서를 섞어 과적합을 줄임 |

### 모델 저장 방식

학습 중 매 에폭마다 검증 데이터의 손실값인 `val_loss`를 확인합니다. `val_loss`가 이전보다 좋아지면 현재 모델을 다음 경로에 저장합니다.

```text
/content/drive/MyDrive/illegal_ad_classifier/best_model.pt
```

카테고리 이름과 숫자 label의 매핑 정보는 다음 파일에 저장합니다.

```text
/content/drive/MyDrive/illegal_ad_classifier/cat2id.json
```

즉, `best_model.pt`는 가장 성능이 좋았던 모델의 실제 가중치이고, `cat2id.json`은 모델 출력 숫자를 사람이 이해할 수 있는 카테고리명으로 바꾸기 위한 사전입니다.

## 9. 평가 방식

학습이 끝나면 `best_model.pt`를 다시 불러와 test set에서 성능을 평가합니다.

평가 결과는 `classification_report`로 출력합니다.

| 지표 | 의미 |
| --- | --- |
| precision | 해당 카테고리라고 예측한 것 중 실제로 맞은 비율 |
| recall | 실제 해당 카테고리인 것 중 모델이 찾아낸 비율 |
| f1-score | precision과 recall을 함께 반영한 점수 |
| support | 해당 카테고리의 실제 샘플 수 |

## 10. 지역명 매핑과 추가 분석

분류 이후에는 게시글의 명사 목록에서 지역명을 찾아 `region_list`를 만듭니다. 이후 코드는 다음 분석을 수행합니다.

| 분석 | 설명 |
| --- | --- |
| 워드 클라우드 | 전체 불법 광고와 마약 관련 단어를 시각적으로 표시 |
| 마약 종류별 지역 분포 | 마약 관련 키워드가 어떤 지역명과 함께 나타나는지 분석 |
| 텔레그램 아이디 추출 | 제목에서 텔레그램 ID 패턴을 찾아 `tg_id`로 저장 |
| 텔레그램 네트워크 분석 | 같은 ID 또는 관련 ID가 여러 게시글에 등장하는 관계를 분석 |
| DB 세부 카테고리 분석 | 특정 사이트나 DB 관련 키워드 빈도 분석 |
| 연락수단 파이차트 | 텔레그램 등 연락수단 노출 비중 시각화 |

## 11. 핵심 요약

이 코드는 다음 세 가지 판단 방식을 섞어서 사용합니다.

1. 사람이 만든 키워드 사전
2. 가벼운 확률 모델인 나이브 베이즈
3. 한국어 딥러닝 모델인 KLUE-RoBERTa

처음에는 키워드 사전으로 비교적 확실한 게시글을 분류하고, 애매한 경우에는 모델의 예측 확률을 참고합니다. 그 후 확실하다고 판단된 데이터만 골라 RoBERTa 모델을 추가 학습합니다.

```text
수집 데이터
  -> 제목 정리
  -> 키워드/지역명 사전 적용
  -> 명사 추출
  -> 규칙 + 나이브 베이즈로 1차 분류
  -> 신뢰도 낮은 데이터는 제외
  -> RoBERTa로 더 강한 분류 모델 학습
  -> 결과 저장 및 시각화
```

# 프로젝트 소개

쌈 채소 농원 '솔바위농원'의 홈페이지에는 2020년 이후 귀농·농사 글이 끊겼다. 대신 상품 Q&A 게시판에는 마약, 프로포폴, 물뽕 같은 불법·유해 게시글이 계속 올라오고 있었다. 이런 **방치된 사이트**는 솔바위농원 하나가 아니었다.

이 프로젝트는 관리가 중단된 웹사이트의 게시판이 어떻게 불법 광고의 유통 창구로 쓰이는지를 데이터로 추적한다. 서로 성격이 다른 65개 방치 사이트에서 게시글을 수집해, **어떤 범죄가 얼마나 성행하는지**, **어떤 텔레그램 아이디가 어느 사이트를 공급망으로 쓰는지**를 분석했다.

- **수집 기간**: 2025년 12월 ~ 2026년 2월 (개발 2/6~2/25, 약 3주)
- **규모**: 65개 사이트 · 수집 게시글 416,006건 · 17개 범죄 카테고리 · 825개 키워드 사전

## 주요 발견

- **약물 게시글은 공급과 수요가 모두 많았고**, 실제로는 마약 유통의 우회로인 경우도 있었다. 전체 약물 게시글의 과반을 올린 텔레그램 아이디의 채팅방을 심층 취재한 결과, 약물보다 마약 거래 안내가 더 많았다.
- **불법공유·핸드폰해킹·약물·취업사기**는 공급(게시글 수) 대비 수요(조회수)가 10배 이상이었다.
- **금융 범죄 게시자가 가장 많았고**, 이들의 게시글은 34개 사이트로 흘러 들어가고 있었다. 개인정보 거래(DB) 범죄는 특정 사이트에 집중되는 등, 사이트마다 대표 범죄 유형이 뚜렷했다.
- **마약 홍보는 수도권과 경남·경북·강원에 집중**됐고, 필로폰(아이스·에페드린) 언급이 가장 많았다.
- 방치 사이트에 올라오는 불법 게시글을 규제할 법적 근거는 아직 공백에 가깝다. 취재 시점 기준 방송미디어통신심의위원회가 마약·도박 등 일부를 서면 심의 대상으로 포함하는 등 제도적 대응이 시작되는 단계였다.

## 데이터 파이프라인

```
사이트 목록 수집 ──▶ 크롤링 ──▶ 전처리 ──▶ 분류 ──▶ 지명 매핑 ──▶ 분석·시각화
   (65개 선정)     (41.6만건)  (특수문자)  (규칙+NB+  (법정동코드)  (워드클라우드·지도·
                                        트랜스포머)               생키·네트워크)
```

### 1. 데이터 수집 (크롤링)
불법 광고가 처음 올라온 사이트를 추적한 뒤 게시글을 재검색해 100여 개 사이트를 확보하고, 그중 2026년에도 게시물이 계속 올라오는 **65개 사이트를 최종 선정**했다. 2026년 게시글만 수집하되 매크로로 게시량이 과도한 경우 사이트당 최대 1만 건으로 제한했다(`is_truncated` 표시). 파일마다 인코딩이 다를 수 있어 `utf-8-sig → cp949 → euc-kr` 순으로 읽기를 시도한 뒤, `site, title, raw_date, date, view, url` 컬럼으로 통일해 병합하고 상세페이지 URL 기준으로 중복을 제거했다. **최종 수집 416,006건.**

### 2. 전처리
검색 회피용 첨자·딩벳 등 특수 유니코드 문자를 `MANUAL_MAP`과 `unicodedata.normalize('NFKC')`로 원형에 가깝게 복원하고, 조회수·날짜 형식을 정제했다(`26-02-13 → 2026-02-13`). 분석 대상이 아닌 러시아어 게시글을 제거해 415,550건으로 정리했다.

### 3. 사전 구축 (키워드 · 지명)
- **키워드 사전**: 불법 유형을 17개 카테고리로 정의하고 총 825개 키워드를 정리했다(금융·도박·유흥·마약·약물·찌라시·문서위조·안락사·디비·짝퉁·취업사기·핸드폰해킹·임신중절·불법공유·중범죄·성인사이트·연락수단). 한 셀에 쉼표로 묶인 키워드를 한 줄에 하나씩 펼치고 NFKC 정규화를 적용했다.
- **지명 사전**: 전국 행정동 리스트(19,832행 · 시도 17 · 시군 158 · 구 77 · 동/면/리 10,362)를 바탕으로 지역명 매핑 사전을 구축했다.

### 4. 형태소 분석 (Kiwi)
`kiwipiepy`의 Kiwi 형태소 분석기 사용자 사전에 키워드·지명을 등록해 중요한 단어가 잘게 쪼개지지 않도록 한 뒤, 제목에서 명사만 추출해 `nouns` 컬럼에 저장했다.

### 5. 카테고리 분류 (규칙 + 나이브 베이즈 + 트랜스포머)
세 가지 판단을 단계적으로 결합했다.

- **1차 — 규칙 + 나이브 베이즈**: 키워드가 뚜렷하면 규칙으로 확정(Tier 1)하고, 여러 카테고리가 겹치면 나이브 베이즈(bigram·카테고리당 2,000개 균형 학습) 예측으로 판정(Tier 2·3)했다. 판별 기준은 `top1_conf ≥ 0.25`, 또는 `top1_conf ≥ 0.18`이고 1·2순위 격차가 0.10 이상일 때. 어느 쪽도 아니면 미분류(Tier 4)로 남겼다.
- **2차 — KLUE-RoBERTa**: Tier 4를 제외한 신뢰도 높은 데이터로 한국어 사전학습 모델 `klue/roberta-base`를 파인튜닝했다(train/val/test 8:1:1, max_len 64, batch 32, lr 2e-5, 조기 종료). 성능 비교를 위해 KoELECTRA도 함께 실험했다. 최적 모델은 `best_model.pt`, 라벨 매핑은 `cat2id.json`으로 저장했다.

### 6. 지명 매핑
분류된 게시글의 명사에서 지역명을 찾아 `region_list`를 만들었다. '남성/고기/안내' 등 혼동 단어는 제외하되 '남성역'처럼 접미사가 붙으면 지역으로 예외 처리하고, 동일 지명이 겹치면 수도권을 우선 매핑했다.

### 주요 컬럼

| 컬럼 | 설명 |
| --- | --- |
| `site` | 게시글이 수집된 사이트명 |
| `title` / `title_cleaned` | 게시글 제목 (원문 / 정제본) |

| `nouns` | 제목에서 추출한 명사 |
| `final_category` | 최종 분류된 범죄 카테고리 |
| `region_list` | 추출된 지역명 목록 |

## 추가로 진행된 사항
